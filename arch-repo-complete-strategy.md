# Arch-Repo Strategy Index

This file remains as a stable entry point, but the full strategy is now split
into smaller documents under `docs/` so the repository is easier to adopt as a
reusable method.

## Why the split

The previous single strategy file mixed:

- method overview
- governance rules
- onboarding guidance
- repository structure
- examples and templates

That was useful as a draft, but it is less effective as reusable method
documentation. The new structure follows the same principle used by successful
method repositories such as `obra/superpowers`: short root orientation, focused
docs for adoption, and smaller guides for operational details.
